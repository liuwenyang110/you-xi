# PRD：农助手适老化交互优化方案
**文档版本**：v1.0  
**编制日期**：2026-04-19  
**负责人**：产品团队  
**目标用户**：60岁及以上，手机使用经验有限的小农户（代表人物：河南农村种粮大爷）  

---

## 一、背景与问题定义

### 1.1 用户画像（JTBD 视角）

| 维度 | 描述 |
|------|------|
| **功能性需要** | 快速叫到一台收割/播种机器，不耽误农时 |
| **情感性需要** | 用手机叫机器，不丢人，不怕被坑 |
| **社会性需要** | 和村里其他人一样用上手机叫机器，有面子 |
| **触发时机** | 庄稼到了节气，眼看别人家都在收，自己还没找到人 |
| **竞争替代方案** | 打电话找熟人、路口等待机器、找村长帮忙 |

### 1.2 当前痛点诊断

根据代码审查（2026-04-19），现有 UI 存在以下高优先级问题：

1. **文案技术化**：`"匹配中"`、`"待协商"`、`"分层派单"`、`"条件校验"` 等词对目标用户毫无意义。
2. **空状态无引导**：`"当前区域暂无匹配机手"` 没有告诉用户下一步该做什么。
3. **字体偏小**：当前基础字号约 24-28rpx，对 60+ 用户视力不友好。
4. **主操作不突出**：关键按钮与次要操作视觉权重相近（见 `contact-confirm.vue`）。
5. **缺乏状态语音反馈**：用户不知道页面有没有在"动"，容易焦虑重复点击。

## 二、四大适老化设计原则

### 原则一：文案大白话（"小学三年级能读懂"标准）

**规则**：
- 禁用词：匹配、派单、候选、协商、确认联系、闭环、同步状态
- 必用替换：

| 禁用词 | 替换词 |
|--------|--------|
| 匹配中 | 正在帮您找机手 |
| 待协商 | 和师傅商量中 |
| 分层派单 | 通知师傅接单 |
| 已建立联系 | 师傅电话拿到了 |
| 待完工确认 | 活干完了，确认一下 |
| 空状态 | 附近暂时没有空闲机手，点下面发个需求等师傅来接 |

**四阶段心智模型**（必须贯穿全产品）：

```
🔍 寻机  →  🚜 路上  →  ⚙️ 作业  →  ✅ 完成
（找机手）  （师傅来了）  （干活中）  （确认完工）
```

---

### 原则二：大字体基准

**基础字号规范**（rpx 单位）：

| 等级 | 标准模式 | 老年模式（elder-mode） | 用途 |
|------|---------|----------------------|------|
| 标题 | 36rpx | **48rpx** | 页面主标题、状态名称 |
| 正文 | 30rpx | **40rpx** | 操作描述、下一步提示 |
| 辅助 | 24rpx | **34rpx** | 时间、编号等次要信息 |
| 按钮 | 30rpx | **42rpx** | 主操作按钮文字 |

**最小触摸区域**：44×44pt（iOS HIG 标准），按钮高度不低于 **96rpx**，老年模式不低于 **120rpx**。

**实施方式**：在 `App.vue` 根节点绑定 `elder-mode` class，所有字号通过 CSS 变量 + `.elder-mode` 选择器覆盖。

```css
/* 示例：设计 token 扩展 */
:root {
  --text-action: 30rpx;
  --btn-height: 96rpx;
}
html.elder-mode {
  --text-base: 40rpx;
  --text-action: 42rpx;
  --btn-height: 120rpx;
}
```

---

### 原则三：高对比色规范（WCAG AA 最低要求）

| 元素 | 现有对比度 | 目标对比度 | 推荐色值 |
|------|-----------|-----------|----------|
| 主操作按钮文字 | ~5:1 | **≥ 4.5:1** ✅ | 白色 `#FFFFFF` on 绿色 `#16a34a` |
| 状态标签文字（pending）| ~3.8:1 ⚠️ | ≥ 4.5:1 | `#7c4f00` on `#fef3c7` |
| 空状态提示文字 | ~2.9:1 ❌ | ≥ 4.5:1 | `#334155` on `#f8fafc` |
| 老年模式正文 | — | ≥ 7:1（AAA） | `#1e293b` on `#f9fafb` |

**具体修改建议**：
- `UxEmpty.vue`：将 `color: var(--text-muted)` 在 elder-mode 下改为 `color: var(--text-primary)`
- 状态 `tone: 'pending'` 的 badge 背景从 `rgba(211,149,22,0.14)` 改为 `#fef3c7`，文字从 `#8a6208` 改为 `#7c4f00`

---

### 原则四：一键操作原则

**核心规则**：完成核心任务不超过 **3 次点击**。

| 场景 | 现有点击数 | 目标点击数 | 优化方案 |
|------|-----------|-----------|---------|
| 发布找机需求 | 首页→填表单→提交 = 3步 ✅ | 3步以内 | 保持，增加一键预填历史作物/亩数 |
| 联系农机主 | 找到机手→确认弹窗→点联系 = 3步 ✅ | 3步以内 | 保持，弹窗文案适老化 |
| 查看订单进度 | Tab→订单页 = 2步 ✅ | 2步以内 | 增加首页"我的订单"快捷入口 |
| 确认完工 | 进入订单详情→点确认 = 2步 ✅ | 2步以内 | 增加首页 push 通知提醒 |

**禁止项**：
- 不允许在核心操作流程中出现"确认是否继续？"的二次弹窗（保留的除外：取消需求时需二次确认）
- 主按钮区域背景色必须与页面有至少 3:1 对比度（防止老人看不出来可点），当前 `contact-confirm.vue` 的 action-panel 已满足

## 三、三大核心场景优化方案

### 场景一：发布需求（发布需求→等机手）

**当前路径**：首页 → 填写需求表单（demand-create.vue）→ 确认发布 → 跳转 matching.vue

**优化点**：

#### A. 首页空状态（已改）
```
优化前：当前区域暂无匹配机手 / 可扩大范围或发布需求等待接单
优化后：附近暂时没有空闲机手 / 点下面的按钮，发个需求等师傅来接
```

#### B. 发布兜底按钮（已改）
```
优化前：✍️ 找不到？发布需求等农机主接单
优化后：点这里发需求，等师傅来接单
```

#### C. 发布中轮询界面（已改）
```
优化前：系统正按距离、机型和服务能力为你寻找合适农机
优化后：正在帮您找附近机手，请稍等 / 我们帮您找附近的机手，您可以放心等候，有消息会通知您
```

**待开发（P1）**：
- [ ] 进入 demand-create.vue 时，自动预填上次的作物类型和亩数
- [ ] 历史需求"一键重发"功能（空状态下显示）

---

### 场景二：联系农机主（师傅接单→拿电话）

**当前路径**：matching.vue 检测到 contact session → 跳转 contact-confirm.vue → 确认 → 拨打

**优化点**：

#### A. 状态徽章（已改）
```
优化前：待确认 / 已建立联系
优化后：师傅等您确认 / 师傅电话拿到了
```

#### B. 主操作按钮（已改）
```
优化前：同意联系并获取号码
优化后：确认联系师傅
拒绝前：暂不联系，继续匹配
优化后：换坚师傅，继续找
```

#### C. Headline（已改）
```
优化前：请确认是否与当前农机主建立联系
优化后：师傅要来了，您确认一下吗？
```

**待开发（P1）**：
- [ ] 接单通知增加**震动+声音**提示（`uni.vibrateLong()`）
- [ ] 联系确认页增加师傅头像、距离、评分大字展示，让老人有安全感
- [ ] 拿到电话后，自动显示"一键拨打"的大按钮（`uni.makePhoneCall`），已经实现，需放大到 elder-mode 高度

**待开发（P2）**：
- [ ] 接单时推送通知，老人不用守着 app

---

### 场景三：查看订单（师傅来了→干活→完成）

**当前路径**：order-detail.vue，轮询 4 秒/次

**优化点**：

#### A. 状态标签（已改）
```
优化前：待协商 / 服务中 / 待完工确认 / 已完成
优化后：和师傅商量中 / 师傅正在干活 / 活干完了，确认一下 / 服务完成
```

#### B. Headline（已改）
```
优化前：双方已建立联系，等待线下协商
优化后：已拿到师傅电话，打电话商量好时间

优化前：等待双方完成完工确认
优化后：师傅说活干完了，确认一下
```

#### C. 进度条文字
```
优化前：履约进度 65%（数字无感知）
优化后：建议增加阶段文字标注："联系好了 → 干活中 → 快完工 → 已完成"
```

**待开发（P1）**：
- [ ] 完工确认的二次弹窗文案改为："师傅说活干完了，您确认后平台会把这次服务标记完成，确认吗？"
- [ ] 增加"联系平台"入口（订单出问题时的逃生口）
- [ ] 重要状态变更时推送本地通知（如状态从"干活中"变为"活干完了"）

## 四、语音提示建议

> 适用于：视力不佳、不识字，或手机小的用户

**实施优先级**：P2 功能，建议 v2.1 版本实现

| 场景 | 建议语音文本 | 触发时机 |
|------|------------|---------|
| 师傅接单 | "有师傅要来帮您干活了，请点确认" | matching.vue 检测到联系单 |
| 拿到电话 | "师傅电话拿到了，可以打电话了" | contact-confirm.vue 确认成功 |
| 师傅开工 | "师傅开始干活了" | 订单状态变为 SERVING |
| 完工提醒 | "活干完了，请您确认一下" | 订单状态变为 FINISHED_PENDING_CONFIRM |
| 服务完成 | "这次服务全部完成了，感谢使用农助手" | 订单状态变为 COMPLETED |

**技术方案**：微信小程序 `wx.createInnerAudioContext()` 播放本地录制音频，或调用文字转语音 API（如阿里云 TTS）。

## 六、验收标准

| 标准 | 验收方式 |
|------|---------|
| 60岁农民无需指导能完成"发需求→确认联系→确认完工"全流程 | 真实用户测试，成功率 ≥ 80% |
| 所有面向用户的状态文案不含"匹配/协商/候选/派单"等技术词 | 代码审查 |
| 主要文字对比度达到 WCAG AA（≥ 4.5:1） | 使用 Contrast Checker 工具验证 |
| 核心任务不超过 3 次点击 | 测试用例验证 |
