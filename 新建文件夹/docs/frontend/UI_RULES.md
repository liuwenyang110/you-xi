# UI 规则

## 中国市场默认方向

- 产品默认面向中国市场，文案、地址、流程、地图能力优先服务中国县乡农业场景。
- 视觉方向采用“务实政企风”：可信、清晰、克制，避免过度炫技表达。
- 默认支持适老化：大字号、高对比、足够点击区域、步骤少、反馈明确。
- 位置服务优先适配高德地图，页面展示应支持乡镇 / 村名称、服务范围、弱网定位降级提示。
- 交互优先适配微信生态心智：手机号登录、短路径操作、卡片式任务入口。

## 状态展示规范

- 农户端和农机主端统一使用大字号、低学习成本表单。
- 关键状态必须使用中文标签，禁止在页面中直接向用户暴露英文状态码。
- 优先使用后端返回的 `statusLabel` 和 `statusDescription` 字段。
- 状态颜色要稳定：
  - 进行中：绿色/蓝绿色
  - 待确认：橙色
  - 已完成：深绿色
  - 失败/取消：灰红或灰色
- 同一状态在移动端和后台必须保持一致的中文表达。

## 首页与主操作

- 首页突出两个主操作：发布需求、登记设备。
- 农户首页优先显示：当前需求状态、最近联系进度、待处理动作。
- 农机主首页优先显示：待响应派单、设备可用状态、服务范围。
- 避免使用“中枢、驾驶舱、引擎”等过度包装词汇，优先用朴实业务语言。

## 表单与地图

- 关键状态使用中文标签，避免抽象英文代码暴露给用户。
- 表单尽量分步、字段少、可预填、可回看。
- 地址字段优先展示乡镇/村庄名称，地图选点要给出明确定位反馈。
- 弱网或地图服务降级时，必须给用户明确提示，不得静默失败。

## 适老化

- 适老化优先，按钮高度和点击区域保持充足。
- 重要信息优先用文字表达，不只依赖颜色或图标。
- 错误提示必须短句直白，避免技术术语。
