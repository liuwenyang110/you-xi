# 农助手 V2 极简适老交互 PRD
**(由第一战区：Claude Sonnet 4.6 产出)**
基于 `@skills/jobs-to-be-done-analyst` 和 `@skills/onboarding-psychologist` 深度挖掘出下沉市场大龄用户的真正诉求。

## 1. 痛点分析与心智模型
- **农户痛点**：看不懂复杂的“撮单阶段”、“状态流转提示”，只需要明确一件事——“农机什么时候来我这”。
- **农机主痛点**：派单不及时，路况不熟，路程时间损耗大。需要绝对的利益导向“听声抢单”。

## 2. 核心优化方案：状态机闭环收敛
我们将从 C端用户 的视角，屏蔽所有复杂的并发锁机制，收敛为最直接的情感化反馈。
订单向农户侧展示的文案严格定为 **四大阶段**：
1. **[配对图标转圈] 正在为您寻找良机...** (对应 Redis GeoHash 调度阶段)
2. **[大卡车开来] 老板接单了，正在路上！** (已绑定 equip_id，锁定阶段)
3. **[拖拉机作业] 老板下地干活中...** (开始业务)
4. **[握手图标] 这次服务完成了！** (完成记录)

## 3. UI 情感化设计承接
- 订单状态流转需配合全局的 Lottie 玻璃拟态动效（交由 Claude Opus 搞定 CSS Tokens及动画注入）。
