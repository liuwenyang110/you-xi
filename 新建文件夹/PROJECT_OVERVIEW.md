# 农助手 V1 - 项目规划与状态文档

## 项目概述

**农助手** 是一个面向中国农村市场的**公益农机联系平台**，以"免费信息对接，不参与交易"为核心原则，帮助农户与农机服务提供者建立直接联系。平台不收取任何费用，不代替双方协商，仅提供信息展示与联系渠道。

> **公益原则**：农户求助信息公开展示，服务者登记设备和联系方式，双方直接沟通，平台完全免费且不介入任何商业行为。

### 当前项目状态
- **项目阶段**：第四阶段 — 公益化重构
- **功能完成度**：核心信息对接流程已实现联调通过
- **重构方向**：从商业撮合模型重构为公益联系平台，移除订单/抽佣/结算逻辑
- **产品方向**：面向中国县乡市场，强调公益信任感、适老大字、微信生态兼容与高德地图定位体验

### 核心业务流程（公益版）
1. 手机号登录（支持测试账号一键登录）
2. 实名认证与角色切换（求助者/服务提供者）
3. 服务者登记农机设备与联系方式，设置服务区域
4. 求助者发布作业需求（类型、位置、时间、面积）
5. 求助者查看、联系合适的服务者（电话直联）
6. 双方自行协商价格和时间，平台不介入
7. 管理后台监控对接统计，维护服务区域

## 技术架构

### 后端技术栈
- **框架**: Spring Boot 3.3.5 + Java 17
- **数据库**: MySQL + Flyway 数据库迁移
- **缓存**: Redis
- **ORM**: MyBatis Plus 3.5.7
- **安全**: Spring Security + JWT
- **API设计**: RESTful API + 统一错误处理
- **地图服务**: 高德地图API集成（带有降级模式）

### 前端技术栈
- **农户/农机主端**: Uni-app (Vue3 + TypeScript + Vite)
- **管理后台**: Vue 3 + TypeScript + Element Plus
- **状态管理**: Pinia
- **构建工具**: Vite
- **代码质量**: ESLint + Prettier + TypeScript
- **部署**: 支持H5、微信小程序、支付宝小程序等多平台

### 已完成的技术优化
1. **统一错误处理**: 建立了完整的异常处理体系
2. **参数校验**: 核心接口均已添加参数验证
3. **数据隔离**: 基于角色的数据权限控制
4. **状态管理**: 统一的枚举和常量定义，并开始补充中文状态标签与说明能力
5. **日志系统**: 请求日志和流程日志基础
6. **前端路由守卫**: 完善的登录和角色校验
7. **中国市场适配基线**: 明确中文状态表达、适老体验与高德地图定位为默认方向

## 样式系统重构计划

### 核心问题
基于之前的分析，当前前端项目存在以下问题：
1. App.vue样式文件过于庞大（530行），难以维护和扩展
2. 设计系统管理分散，缺乏统一的样式变量管理
3. 页面样式存在冗余和重复代码
4. 性能优化不足，缺乏按需加载和懒加载机制
5. 设计系统扩展性差，难以支持新主题或品牌定制

### 优化目标
1. 重构样式架构，实现模块化管理
2. 建立集中式设计系统（设计令牌）
3. 提升页面加载性能和用户体验
4. 增强设计系统的可维护性和扩展性
5. 统一移动端、后台端、后端接口中的中文状态表达与中国市场体验基线

### 技术架构设计

#### 分层架构
设计令牌层 → 组件样式层 → 页面样式层 → 主题系统层


#### 核心模块划分
1. **design-tokens**: 设计令牌定义（CSS变量、TypeScript类型）
2. **core-styles**: 核心样式（重置、基础、工具类）
3. **component-styles**: 组件样式模块
4. **theme-system**: 主题管理系统
5. **optimization**: 性能优化模块（样式压缩、懒加载）

#### 数据流
用户选择主题 → 主题系统切换 → 设计令牌更新 → 组件样式重新计算 → 页面重新渲染
